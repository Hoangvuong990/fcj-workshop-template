**On this page**, you will need to introduce your worklog. **How** did you complete it? How many weeks did you take to complete the program? **What** did you do in those weeks?

Typically, and as a standard, a worklog is carried out over about 3 months (throughout the internship period) with weekly contents as follows:

**Week 1:** [Getting familiar with AWS and basic AWS services](1.1-week1/)

**Week 2:** [AWS Cost Management and AWS Budgets setup](1.2-week2/)

**Week 3:** [AWS Support and Networking Fundamentals (VPC, NACL, Security Group, ELB)](1.3-week3/)

**Week 4:** [Amazon EC2 Core Concepts, EBS, Auto Scaling, and Load Balancers](1.4-week4/)

**Week 5:** [Amazon S3 Storage, Hybrid Storage, Shared Responsibility, and AWS IAM](1.5-week5/)

**Week 6:** [AWS Databases (RDS, Aurora, Redshift), AWS Organizations, and Security Hub](1.6-week6/)

**Week 7:** [Wakan Kickoff, Workspace Setup, and UI Wireframing on Google Drawings/Jamboard](1.7-week7/)

**Week 8:** [API Contract Alignment, Static UI Components, and Amazon Cognito Integration](1.8-week8/)

**Week 9:** [UI Selection Flow Completion, API Gateway Integration, and Error/Loading States](1.9-week9/)

**Week 10:** [Timeline Layout, Map Integration, Mobile Responsiveness, and E2E Testing](1.10-week10/)

**Week 11:** [UI/UX Polishing, Interface Testing, and Bug Fixing](1.11-week11/)

**Week 12:** [System Integration Dry Run, Post-Test Optimization, and Technical Report Compilation](1.12-week12/)
